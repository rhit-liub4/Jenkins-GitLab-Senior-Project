STEP 1

Our first goal was to become familiar with the GitLab Branch Source Plugin, its implementation of dependencies, and where our feature would best be implemented. ​

STEP 2

Through this process, we found the feature could not be implemented at a GitLab-specific level, so we decided to research both the top-level abstract implementation as well as the implementation in the Jenkins Git plugin, where the feature was eventually added.​

STEP 3

Once we created and implemented our feature, there was a set of testing and linting procedures set out by the maintainers of Jenkins – to homogenize code style and ensure proper testing before submitting a pull request.​

FINISH

After a submission for approval, we received comments with several requests for changing of code style and test completion. After making fixes and submitting another request, our changes were merged in.​
